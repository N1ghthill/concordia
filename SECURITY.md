# Politica de Seguranca

A seguranca do Concordia e tratada como requisito de engenharia, nao como etapa final.

## Versoes com suporte

| Versao | Suporte de seguranca |
| --- | --- |
| `main` | Ativo |
| `pre-release` | Ativo |
| Versoes antigas | Encerrado |

## Como reportar vulnerabilidades

Nao reporte vulnerabilidades em issues publicas.

Use o fluxo privado de seguranca do repositorio no GitHub (Security Advisory) com:

- Descricao do problema.
- Impacto esperado.
- Passos para reproducao.
- Possivel mitigacao.

## Processo de resposta

- Triagem inicial em ate 72 horas.
- Confirmacao tecnica e classificacao de severidade.
- Correcao priorizada conforme risco.
- Divulgacao responsavel apos mitigacao.

## Boas praticas esperadas

- Principio do menor privilegio.
- Revisao de mudancas sensiveis por mais de uma pessoa.
- Registro claro de decisoes de seguranca.
- Auditoria continua de riscos arquiteturais.
